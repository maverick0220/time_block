# 2026-04-16 工作记录

## 多客户端同步数据不互通问题修复

### 问题根因（三处）
1. **B 客户端 uploadRange 起点错误**：从未同步时 `fallbackStartDate = null`，`calcUploadRange` 直接用"今天"作为起点，导致 B 的 uploadRange 完全不覆盖 A 上传的历史数据
2. **服务端 find_missing_for_client 回填逻辑不完整**：只回填「客户端未上传的天」，漏掉「客户端已上传但合并后服务端有更多事件」的场景（A+B 同天不同时间段，B 拿不到 A 的那部分）
3. **补丁写入 Hive 后内存未更新**：`dayRecords` 内存 Map 不刷新，用户看不到同步来的新数据

### 修复方案
| 文件 | 修改内容 |
|------|------|
| `lib/network/dataSync.dart` | `runFullSync` 首次同步时自动计算 fallback：取 Hive 最早有事件的日期，若 Hive 空则用当年 1 月 1 日；`SyncResult` 新增 `patchedDates` 字段 |
| `server/merger.py` | `find_missing_for_client` 新增第二条回填规则：客户端已上传但合并后事件数更多的天也回填；新增 `client_day_records` 参数 |
| `server/app.py` | 调用 `find_missing_for_client` 时传入 `client_day_records` |
| `lib/loaders.dart` | `UserProfileLoader` 新增 `applyPatchedDates` 方法，从 Hive 重读补丁日期数据并 notifyListeners |
| `lib/view/appBarViiew.dart` | 同步后改用 `userProfileLoader.applyPatchedDates(result.patchedDates)` 精确刷新内存 |

## Dashboard 可视化 Web 页面开发（2026-04-16）

### 新增文件
| 文件 | 说明 |
|------|------|
| `server/dashboard.html` | 纯前端单文件 Dashboard，通过 `/api/dashboard-data` 获取数据，深黑主题 |
| `server/app.py` | 新增 `GET /dashboard` 路由（返回 dashboard.html）和 `GET /api/dashboard-data` 路由（返回全量可视化数据） |

### 功能说明
- **左侧**：瀑布流式日历（每行一周，月份标注左侧），色块状态：绿=完整覆盖、黄=有冲突、蓝=有数据、黑=无数据
- **右侧**：点击日期后展开当天24小时×4格的时间块视图，颜色与客户端完全一致
- Hover 有 tooltip 显示事件名 / 时间段 / 备注
- 底部统计栏：完整天 / 冲突天 / 部分天 / 总记录数

### 访问方式
启动服务端后，浏览器访问 `http://<IP>:5001/dashboard`
