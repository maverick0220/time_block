# 2026-04-18 工作记录

## 服务端持久化层迁移：JSON → SQLite

### 修改文件
| 文件 | 改动 |
|------|------|
| `server/storage.py` | 完全重写，JSON 文件读写 → SQLite；保留所有旧接口签名（load_main / save_main / load_conflicts / append_conflict / load_version_info / bump_version），同时新增细粒度接口 |
| `server/app.py` | 改用细粒度接口（按需加载，不再全量读写），/sync 只加载涉及日期的数据 |
| `server/migrate.py` | 新增一次性迁移脚本，将旧 JSON 迁移到 SQLite，自动备份 JSON 到 data/json_backup/ |

### SQLite 数据库结构（data/timeblock.db）
| 表名 | 内容 |
|------|------|
| event_info | 事件类型（name UNIQUE，含 color_r/g/b、belonging_to、sort_order）|
| day_records | 主时间块数据（date_key PRIMARY KEY，events_json TEXT）|
| conflicts | 冲突记录（每条一行，含 date_key / client_id / events_json）|
| version | 版本历史（保留最近 200 条）|
| meta | 全局 key-value 元信息（如 last_updated）|

### 性能改进
- /sync 接口：只加载 client 上传的日期 + uploadRange 内服务端有的日期，不再全量读取
- WAL 模式（journal_mode=WAL）：并发读写性能提升，崩溃安全

### 数据迁移
```bash
cd server/
python3 migrate.py            # 迁移（保留旧 JSON）
python3 migrate.py --delete-json  # 迁移后删除旧 JSON
```
