# 2026-04-23 工作记录

## 四个 Bug 修复与优化

### 1. 同步逻辑优化（syncConfig.dart + dataSync.dart）
- **原问题**：`calcUploadRange` 以"上次截止日期+1天"为起点，同一天同步过一次后 `lastSyncEndDate=今天`，下次 startDate > today → 返回 null → 提示"已是最新"
- **修复**：
  - 新增 `lastSyncTimestamp`（ISO8601）记录上次同步时刻
  - `calcUploadRange` 改为：startDate = 上次同步日期（不再+1），endDate = 今天；始终包含今天
  - 同步成功后同时更新 `lastSyncEndDate` 和 `lastSyncTimestamp`
  - 移除"已是最新无需同步"的提前返回路径

### 2. B客户端获取A客户端数据问题
- **根因**：同上，今天已同步后 range 为 null，不再上传/下载任何内容，导致补丁也收不到
- **修复**：每次同步都包含今天，服务端返回的 patch 中若有其他设备同步的数据，客户端会正常写入 Hive

### 3. AppBar 选中块事件名显示优化（loaders.dart + appBarViiew.dart）
- **原问题**：只在所有选中色块类型相同时显示，多类型时直接置空
- **修复**：
  - `loaders.dart`：按时间顺序遍历 selectedBlocks，合并相邻相同项，用 ` · ` 拼接多个事件名
  - `appBarViiew.dart`：文字区域改为 `Expanded`，字号 11，`maxLines: 2`，`ellipsis` 溢出处理；同时 `listen: true` 以响应实时变化

### 4. EditPage EventInfo 拖动手柄间距（eventInfoEditView.dart）
- **修复**：删除/编辑按钮图标缩小至 18px，`SizedBox(width: 32)` 在删除按钮右侧留出拖动手柄空间
- 按钮 padding 设为 zero，避免占用过多行高

## 第二轮优化（同日下午）

### 5. 服务端新增 sync_log 同步操作日志（server/storage.py + server/app.py）
- **storage.py**：新增 `sync_log` 表（字段：client_id、client_time、server_time、range_start/end、uploaded_dates、merged_days、conflict_days、patch_days、new_version），最多保留 1000 条
- 新增函数：`append_sync_log(entry)`、`get_sync_log(limit, client_id)`、`get_sync_log_summary()`
- **app.py**：`/sync` 路由末尾调用 `append_sync_log` 记录每次同步；新增 `/sync-log` 接口（支持 `?limit=N&client=ID&summary=1` 参数）

### 6. 修复 cancelSelection 后 selectedBlockEvent 不清空（loaders.dart）
- **原问题**：`cancelSelection()` 只 clear selectedBlocks 但没有清空 `selectedBlockEvent`，导致 cancel 后 AppBar 顶部仍显示旧事件名
- **修复**：在 `selectedBlocks.clear()` 之后补加 `selectedBlockEvent = ""`，与 `wipeSelection` / `recordBlocksAsEvent` 保持一致
