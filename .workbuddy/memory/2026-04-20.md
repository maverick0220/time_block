## AnalyzeView 数据可视化功能（2026-04-20）

### 新增依赖
- `fl_chart`：Flutter 图表库，通过 `dart pub add fl_chart` 添加

### 新增文件
| 文件 | 说明 |
|------|------|
| `lib/view/analyzeView.dart` | 数据分析可视化页面，完整实现三种图表+筛选控件 |

### 修改文件
- `main.dart`：第三个 Tab 从 `Center(child: Text('analys'))` 替换为 `AnalyzeView`，新增 import

### 功能说明
**页面布局（从上到下）**
1. 时间段选择器：调用系统 `showDateRangePicker`，显示所选范围和天数
2. 事项多选弹窗入口：底部 Sheet，支持全选/全不选，确认后重新计算
3. 三分类切换器：折线图 / 饼图 / 横向柱状图，带动画切换
4. 图表视图区域（占 Expanded）：支持水平滑动（手势 >40px）切换同等时长的前后时间段

**数据层 AnalyzeData**
- 从 Hive `dayRecords` 按日期范围和 selectedEvents 过滤
- 每个 event 每天 block 数 × 15 = 分钟数
- totalMinutes、grandTotal 方便各图表调用

**三种图表**
| 图表 | 特点 |
|------|------|
| 折线图 | 每 eventInfo 一条折线，x=日期，y=分钟数；超60天自动降采样；左轴显示 h/m |
| 饼图 | 全时段各 eventInfo 总时长比例；触摸显示名称+时长 |
| 横向柱状图 | 按总时长降序，柱长相对最大值等比；带动画 |

**底部图例**：所有图表统一在底部显示 eventInfo 颜色点 + 名称 + 总时长

## AnalyzeView Bug 修复（2026-04-20 晚）

### 问题1：时间范围选择器报错
- **原因**：`showDateRangePicker` 传 `locale: const Locale('zh', 'CN')` 需要 `flutter_localizations`，项目未添加该依赖
- **修复**：改为自定义 `_DateRangePickerDialog`（Dialog 形式，深色主题）
  - 支持点击选择开始/结束日期（两步法），范围高亮显示
  - 月份导航前进/后退，今天有黄色边框标记

### 问题2：横向柱状图 Row overflow
- **原因**：`barWidth = constraints.maxWidth - 80` 未扣除时间标签宽度和 SizedBox(6) 间距，导致总宽超出
- **修复**：固定 `nameColW=64`、`timeColW=44`，柱子区用 `Expanded` 包裹，`barWidth` 基于 `Expanded` 内实际可用宽计算，彻底杜绝溢出
